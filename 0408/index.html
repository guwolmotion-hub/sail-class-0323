구도 이론

구도 잡기 — 1/3 구도 & 중심 구도

좋은 화면을 만드는 첫 번째 단계는 구도입니다. 카메라를 세팅하기 전에 어떤 구도로 찍을지 결정하는 것이 중요합니다.

⅓

1/3 구도 (Rule of Thirds)

화면을 3×3 그리드로 나누고, 주요 피사체나 시선을 그리드 교차점(황금점)에 배치하는 구도법입니다.

  • C4D 뷰포트: Shift+V → Grid 옵션에서 3×3 활성화
  • 자연스럽고 역동적인 느낌 연출에 유리
  • 영상·사진에서 가장 많이 쓰이는 기본 구도
🎯

중심 구도 (Center Composition)

피사체를 화면 정중앙에 배치하는 구도법입니다.

  • 안정감·대칭미 강조
  • 제품 광고, 포스터, 타이틀 시퀀스에 자주 사용
  • 단조로울 수 있으므로 배경 디자인으로 변화를 줄 것
실무 팁 카메라 Xpresso를 세팅하기 전에 레퍼런스를 보며 어떤 구도를 쓸지 먼저 결정하세요. 구도가 정해지면 나머지 파라미터 값이 자연스럽게 따라옵니다.

파라미터 상세

각 파라미터 역할 & 활용법

10개의 유저 데이터가 각각 어떤 시각적 변화를 만드는지 이해하면 카메라 연출의 폭이 훨씬 넓어집니다.

📏 Distance

카메라가 이동하는 Circle의 반지름을 제어. 값이 클수록 피사체에서 멀어짐

Circle → Object → Radius
🔭 Focal Length

렌즈 화각 조절. 값이 클수록 망원(줌인), 작을수록 광각(와이드)

Camera → Object → Focal Length
↕️ Height

카메라 경로(Circle)의 높이(Y축)를 제어. 부감/앙각 구도 전환

Circle → Coord → Position Y
🔄 Rotation From Center

피사체를 중심으로 카메라가 수평 회전하는 각도 제어 (Turntable 무빙)

Circle → Coord → Rotation H
↗️ Tilt

카메라 자체의 기울기(롤) 조절. 화면을 비스듬히 기울여 역동적인 느낌 연출

Camera → Coord → Rotation B
🌀 Aperture

옥테인 카메라의 조리개 값. 값이 클수록 피사계심도(DoF) 효과가 강해짐

Octane Camera Tag → ThinLens → DoF → Aperture
🎯 Target Object

카메라가 항상 바라볼 목표 오브젝트 지정. 카메라가 어디 있든 피사체를 향함

Target Tag → Target Object
🔍 Focus Object

DoF(피사계심도) 포커스가 맞춰질 오브젝트 지정. Aperture와 함께 사용

Camera → Object → Focus Object
🔗 Target Constraint

Constraint Tag의 PSR 방식으로 카메라 방향을 특정 오브젝트에 고정

Constraint Tag → PSR → Target
📌 Focus Constraint

DoF 포커스 대상을 Constraint Tag로 연결. 움직이는 오브젝트 추적 포커싱 가능

Constraint Tag → PSR → Target
구조 이해 Target Tag와 Constraint Tag는 유사하지만 역할이 다릅니다. Target Tag는 카메라가 항상 피사체를 향하게 하고, Constraint Tag는 위치·회전·스케일(PSR) 전체를 다른 오브젝트에 종속시킵니다.
가렌더 작업

애니메이션 가렌더 (Pre-render) 작업 흐름

최종 렌더 전에 전체 애니메이션 흐름을 빠르게 확인하는 가렌더 작업입니다. 뷰포트 렌더러로 빠르게 확인하고 구성을 점검합니다.

가렌더 작업 순서

1
씬별로 C4D 파일 분리 저장

씬(장면)마다 별도의 `.c4d` 파일로 저장. 파일명에 씬 번호/이름 명시 (예: `scene_01_intro.c4d`)

2
렌더 모드 → 뷰포트 렌더러 설정

렌더 설정(Ctrl+B) → Output에서 렌더러를 Viewport Renderer로 변경. 옥테인 렌더 대비 속도가 빠름

3
저장 경로 설정

Save 탭 → Output File에 저장 폴더와 파일명 지정. 시퀀스 이미지(PNG/JPG) 또는 영상 포맷 선택

4
저장 프레임 범위 설정

Output 탭 → Frame Range에서 시작~끝 프레임 확인. 프레임 레이트(FPS)도 맞는지 체크

5
렌더링 시작

Shift+R 또는 렌더 메뉴 → Render to Picture Viewer로 렌더링 실행

⚡

뷰포트 렌더러의 장점

  • 옥테인 대비 렌더 속도가 매우 빠름
  • 애니메이션 타이밍·구도 검토에 적합
  • 클라이언트 확인용 프리뷰로 활용 가능
📁

씬 분리 저장의 이유

  • 파일이 가벼워져 작업 속도 향상
  • 씬별 독립 수정이 용이
  • 팀 작업 시 파일 충돌 방지
  • 렌더 큐 관리가 명확해짐
