개요

다이나믹 시뮬레이션 — C4D의 물리 엔진

다이나믹 시뮬레이션은 C4D 내에서 중력, 충돌, 반발력 등 물리 법칙을 계산해 오브젝트를 현실처럼 움직이게 하는 기능입니다. 키프레임 없이도 자동으로 자연스러운 물리 움직임을 만들 수 있습니다.

⚠
버그가 많다 — 이게 핵심
컴퓨터가 물리를 계산하는 방식 특성상 버그가 자주 발생합니다. 다이나믹 수업의 진짜 목적은 기능을 외우는 것이 아니라, 오류를 찾아서 고치는 방법을 익히는 것입니다.
💡
왜 전체 기능을 알아야 하나?
버그가 생겼을 때 원인을 파악하려면 시뮬레이션 내 모든 기능이 어떤 역할을 하는지 알고 있어야 합니다. 하나라도 모르면 어디서 문제가 생겼는지 추적이 불가능합니다.
바디 타입

다이나믹 바디 태그 4종 — 역할 구분이 핵심

다이나믹 시뮬레이션은 오브젝트에 태그를 붙여서 역할을 지정하는 방식으로 작동합니다. 어떤 태그를 붙이느냐에 따라 오브젝트의 물리적 성질이 완전히 달라집니다.

Rigid Body
🧱
리지드 바디
딱딱한 물체
오브젝트를 중력의 영향을 받는 충돌체로 변환합니다. 실제로 움직이면서 다른 오브젝트와 부딪혀야 하는 오브젝트에 적용합니다. 모양이 변형되지 않는 단단한 물체를 시뮬레이션합니다.
예: 공, 박스, 타이포그래피 글자
Soft Body
🫧
소프트 바디
부드러운 물체
충돌 시 오브젝트의 형태 자체가 변형되는 시뮬레이션입니다. 젤리, 쿠션, 풍선처럼 충격을 받으면 찌그러지고 튕겨나오는 부드러운 재질의 물체를 표현할 때 사용합니다.
예: 젤리, 풍선, 쿠션, 점액
Collider Body
🏗
콜라이더 바디
고정된 충돌벽
오브젝트를 중력의 영향을 받지 않는 충돌체로 변환합니다. 중력으로 떨어지지 않고 제자리에 고정된 채 다른 오브젝트의 충돌을 받아냅니다. 씬의 바닥이나 벽면에 적용합니다.
예: 바닥, 벽, 고정된 장애물
Ghost Body
👻
고스트 바디
투명 충돌체
시뮬레이션에서는 존재하지만 렌더링에는 보이지 않는 충돌체입니다. 보이지 않는 벽이나 트리거 영역을 만들 때 사용합니다. Dynamic 항목을 Ghost로 설정하면 됩니다.
예: 보이지 않는 경계선, 충돌 트리거 영역
Dynamic 항목 On/Off/Ghost — 한 태그로 3가지 상태 전환
On→ Rigid Body (중력 영향 O, 충돌 O)
Off→ Collider Body (중력 영향 X, 충돌 받기만)
Ghost→ Ghost Body (보이지 않는 충돌체)
Dynamics 탭

Dynamics 항목 — 시뮬레이션 상태와 시작 조건 설정

Set / Clear Initial State
시뮬레이션 시 오브젝트의 초기 위치값을 고정(Set)하거나 초기화(Clear)하는 기능입니다. 원하는 시점에서 시뮬레이션이 시작되도록 기준점을 잡아줄 때 사용합니다.
⚠ Clear Initial State는 일반 단독 오브젝트에서는 작동하지 않습니다. 클로너(Cloner) 또는 Parent 관계로 묶인 오브젝트에서만 사용 가능합니다.
Trigger — 시뮬레이션 시작 조건
시뮬레이션을 언제, 어떤 조건에서 시작할지 결정하는 항목입니다. 같은 씬이라도 Trigger 설정에 따라 결과가 완전히 달라집니다.

Collision 항목 — 계층 구조 & 개별 요소 설정

Inherit Tag — Parent 관계에서의 시뮬레이션 방식
여러 오브젝트가 Parent(부모) 관계로 묶여 있을 때, 시뮬레이션을 어떤 방식으로 적용할지 결정합니다.
Apply Tag To Children
자식 오브젝트 하나하나 전부 개별적으로 시뮬레이션 적용. 각 오브젝트가 독립적으로 움직입니다.
Compound Collision Shape
Parent로 묶인 모든 오브젝트를 하나의 오브젝트로 인식해서 시뮬레이션. 전체가 한 덩어리로 움직입니다.
None
태그가 걸린 최상단 오브젝트만 인식. 자식은 무시합니다.

Shape 설정

Collision Shape — 충돌 계산 정밀도 설정

Shape는 오브젝트의 형태를 물리 엔진이 얼마나 정교하게 계산할지 결정합니다. 정밀할수록 현실적이지만 연산 부하가 커집니다. 상황에 맞게 선택하는 것이 중요합니다.

낮음
Automatic
오브젝트 형태를 가장 단순하게 계산합니다. 박스, 구, 캡슐 등 단순한 형태로 근사합니다. 연산 부하 최소.
중간
Automatic (MoDynamics) ★ 자주 사용
기본 Automatic보다 훨씬 정교하게 계산하지만, 시뮬레이션 오브젝트가 서로 겹치는 상황에서는 제대로 작동하지 않을 수 있습니다. 복잡한 형태의 단일 충돌에서 주로 사용합니다.
높음
Moving Mesh
가장 정교하게 오브젝트 형태를 계산합니다. 겹쳐지는 시뮬레이션(Soft Body, 복잡한 충돌)에서도 정상 작동합니다. 연산 부하가 크므로 꼭 필요한 경우에만 사용합니다.
정적
Static Mesh 콜라이더 바디 전용 추천
가장 정교하지만 움직이지 않는 오브젝트 전용입니다. 움직임 계산이 없어 연산 비용이 없으므로 콜라이더 바디(바닥, 벽)에 적용하면 가성비 최고입니다.
최적화
Another Object ★★★ 최적화 핵심
오브젝트의 충돌 형태를 다른 오브젝트의 형태로 대체해서 계산합니다. 복잡한 고폴리 오브젝트의 충돌을 단순한 저폴리 프록시 메시로 처리해 시뮬레이션 최적화에 사용하는 고급 기법입니다.
사용법: 복잡한 오브젝트 옆에 단순한 대체 메시를 만들고, Another Object 항목에 대체 메시를 지정 → 충돌 계산은 단순 메시 기준으로 처리
시뮬레이션 최적화

시뮬레이션 최적화 — 오브젝트 면 상태가 핵심

다이나믹 시뮬레이션이 제대로 작동하려면 오브젝트 자체의 메시 상태가 중요합니다. 아무리 설정이 맞아도 메시에 문제가 있으면 시뮬레이션이 오작동합니다.

✗ 면이 너무 많으면 안 됩니다
고폴리곤 메시는 충돌 계산량이 폭증합니다. 시뮬레이션용 오브젝트는 최대한 적은 폴리곤으로 구성하고, 고폴리 디테일이 필요하면 Another Object로 저폴리 대체 메시를 지정합니다.
✗ N-Gon(엔곤)이 있으면 안 됩니다
N-Gon은 4개 이상의 꼭짓점을 가진 다각형 면으로, 삼각화(triangulate) 과정에서 예측 불가능한 결과를 낼 수 있습니다. 시뮬레이션 전 메시에 N-Gon이 없는지 반드시 확인하고, 있다면 삼각형 또는 사각형 폴리곤으로 정리합니다.
시뮬레이션 오류가 생겼을 때 체크리스트
①메시에 N-Gon 있는지 확인
②폴리곤 수가 과도하게 많지 않은지 확인
③Shape 설정이 오브젝트 특성에 맞는지 확인
④Trigger 조건이 의도와 맞게 설정되어 있는지 확인
⑤Parent 관계가 있다면 Inherit Tag 옵션 확인
⑥콜라이더 바디(바닥/벽)가 제대로 설정되어 있는지 확인
