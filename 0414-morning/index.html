Xpresso 심화

공 굴리기 Xpresso — 왜 필요한가?

공이 이동할 때 Position만 애니메이션하면 공이 미끄러지듯 움직입니다. 실제로 굴러가는 것처럼 보이려면 이동 거리에 맞게 회전도 자동으로 계산되어야 합니다.

❌ 문제

Position만 키프레임 → 공이 미끄러지듯 이동. 바퀴가 돌지 않는 느낌.

→
✓ 해결

Xpresso로 Position과 Rotation을 원의 둘레 공식으로 자동 연동. 이동 거리에 비례해 회전각도 계산.

01
User Data로 0~100% 컨트롤러 생성

퍼센트 단위로 동작을 제어하는 마스터 슬라이더. 0%=시작, 100%=끝 위치.

02
이동 거리 설정 (내 마음대로)

공이 0~100% 사이에 몇 cm를 이동할지는 디자이너가 직접 결정. Range Mapper로 매핑.

03
회전각도는 이동 거리 + 공 반지름으로 계산

이동 거리가 결정되면 회전은 공식으로 자동 계산됨. 원의 둘레 = 2 × π × 반지름.

핵심 수식

원의 둘레 공식 — 회전각도 계산 원리

공은 자신의 둘레 길이만큼 이동했을 때 정확히 한 바퀴(360°)를 회전합니다. 이 물리 법칙을 Xpresso에 그대로 구현합니다.

원의 둘레
C = 2 × π × r
C = 둘레 길이 π ≈ 3.14159 r = 공의 반지름
→
회전각도
θ = (d ÷ C) × 360°
θ = 회전각도 d = 이동 거리 C = 공의 둘레
예시 계산
공 반지름 50 cm
둘레 (2×π×50) ≈ 314 cm
이동 거리 314 cm
회전 정확히 360°
핵심 이동 거리 ÷ 둘레 = 회전 비율. Xpresso에서는 이 계산을 Range Mapper + Math 노드가 자동으로 처리합니다. 반지름(Radius)이 달라지면 둘레가 달라지므로 Sphere 오브젝트의 Radius 값을 노드에 직접 연결해야 합니다.

User Data (퍼센트 컨트롤러) — 오브젝트에 추가한 슬라이더. 0~100으로 설정해 전체 동작의 마스터 역할.

Range Mapper — 입력 범위(0~100)를 원하는 출력 범위(이동거리 or 회전각도)로 변환하는 핵심 노드.

Constant(2×π) × Sphere Radius — 원의 둘레 공식(2πr)을 노드로 구현. Radius가 바뀌면 자동으로 둘레도 바뀜.

취업 & 포트폴리오

포트폴리오 & 취업 현실 팁

실력은 반복 작업으로 늘어납니다. 과제를 잘 모아두면 그 자체가 포트폴리오가 됩니다.

📌
많이 해야 느니까 — 과제를 성실하게

실력은 이론이 아니라 반복 작업으로 쌓입니다. 과제 하나하나가 실력이 되고 습관이 됩니다.

💼
과제 결과물을 잘 모아두면 그것이 포트폴리오

수업 중 만든 작업물도 정리해서 모아두면 취업 시 바로 쓸 수 있는 포트폴리오가 됩니다. 버리지 말고 아카이빙하세요.

🎯
면접에서 자주 나오는 질문

① "인천 사는데 출퇴근 괜찮아요?" → 이동 시간에 대한 의지를 확인하는 질문. 자신감 있게 답할 것.
② "공부하면서 만든 작업물 있어요?" → 과제와 개인 작업물을 미리 정리해두면 바로 보여줄 수 있음.
